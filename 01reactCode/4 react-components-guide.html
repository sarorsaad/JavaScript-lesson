Component Syntax

Simple React Components

  • ✓ Can render static HTML markup
  • ✓ Can render based on JSON data
  • ✓ Can be nested for complex layouts
  • ✓ Can use props from parent components

Complex React Components

  • ✓ Include advanced rendering logic
  • ✓ Can maintain internal state
  • ✓ Can use lifecycle methods
  • ✓ Can include custom event handlers

Important JSX Rules

  • ⚠️ Components must return only ONE root element
  • ⚠️ All HTML markup must be wrapped in a container (e.g., div)
  • ⚠️ JSX makes writing HTML in React more elegant

Types of React Components

Function Components (Stateless)

Definition: JavaScript functions that accept props and return React elements (JSX).

Characteristics:
  • Simple and lightweight
  • Initially known as "Stateless Components"
  • Can now use React Hooks (useState, useEffect) for state management

Use Cases: Ideal for simple UI elements or components with minimal logic.

function Welcome(props) { const [isActive, setIsActive] = useState(false); // Using Hooks return

Hello, {props.name}

; }

Class Components (Stateful)

Definition: ES6 classes that extend React.Component with a required render() method.

Characteristics:
  • Can manage state using this.state property
  • Access to lifecycle methods (componentDidMount, etc.)
  • More verbose syntax compared to function components

Use Cases: Legacy applications or complex state management scenarios.

class Welcome extends React.Component { state = { isActive: false }; render() { return

Hello, {this.props.name}

; } }

Modern Perspective

The distinction between stateless and stateful components is now largely historical. Since React 16.8 introduced Hooks:

  • Function components can handle both state and lifecycle methods
  • Class components are maintained for legacy support
  • Best Practice: Use function components with hooks for new development

Component Communication & Modular Design

Modular Design Pattern in React

React promotes a modular design pattern through its export and import system:

// ContentA.js const ContentA = () => { return
Content A Component
; } export default ContentA; // index.js import ContentA from './ContentA'; import Container from './Container';
Key Benefits:
  • Code Reusability: Components can be reused across the application
  • Separation of Concerns: Each file handles a single responsibility
  • Scalability: Easy to maintain and scale as the application grows
  • Better Testing: Isolated components are easier to test

Note: React supports both default exports (export default) and named exports (export const). Named exports allow multiple exports from a single file.

Understanding Props

Interactive Props Example: Image Slider

Step 1: Why We Need Props

Problem: Inflexible Component


// Original inflexible component
function HelloWorld() {
    return <span>Hello, World!</span>;
}

// Can only output: Hello, World!
<HelloWorld />

Solution with Props:


function HelloWorld(props) {
    return <span>Hello, {props.name}!</span>;
}

// Now we can reuse it!
<HelloWorld name="World" />    // Output: Hello, World!
<HelloWorld name="John" />     // Output: Hello, John!
<HelloWorld name="Students" /> // Output: Hello, Students!

Step 2: Values of Props

Different Types of Props:


// String literals
<MyComponent prop="My String Value" />

// Template literals
<MyComponent prop={`My String Value ${myVariable}`} />

// Number literals
<MyComponent prop={42} />

// Boolean literals
<MyComponent prop={false} />

// Object literals
<MyComponent prop={{ property: 'Value' }} />

// Array literals
<MyComponent prop={['Item 1', 'Item 2']} />

// JSX
<MyComponent prop={<Message who="Joker" />} />

// Variables
<MyComponent prop={myVariable} />

Step 3: Passing Down Props (Props Drilling)


// Child Component
function Hello({ who }) {
    return <div>Hello, {who}!</div>;
}

// Parent Component
function HelloPeople({ persons }) {
    return (
        <div>
            {persons.map((person, index) => {
                return <Hello who={person} key={index} />;
            })}
        </div>
    );
}

// Usage
<HelloPeople persons={['Joker', 'Batman']} />

Step 4: Special Props in React

1. The children Prop

The children prop is automatically passed by React to components and contains whatever is placed between the opening and closing tags of a component.


// Parent Component
function Parent({ children }) {
    console.log(children); // logs <span>I'm a child!</span>
    return <div>{children}</div>;
}

// Usage
<Parent>
    <span>I'm a child!</span>
</Parent>

// Output
<div><span>I'm a child!</span></div>
Key Points about children:
  • It's automatically handled by React
  • Allows component composition
  • Can contain any valid JSX
  • Makes components more flexible and reusable

2. The key Prop

The key prop is used when rendering lists of elements to help React track changes efficiently.


function HelloPeople({ persons }) {
    return (
        <div>
            {persons.map((person, index) => {
                return <Hello who={person} key={index} />;
            })}
        </div>
    );
}

// Better approach with unique IDs
persons.map(person => (
    <Hello who={person.name} key={person.id} />
))
Key Points about key:
  • Must be unique among siblings
  • Helps React identify which items have changed
  • Important for performance optimization
  • Usually uses unique IDs or indices (though indices aren't always ideal)
Best Practices:

For children:

  • Use to create wrapper components
  • Useful for layout components
  • Makes components more flexible

For keys:

  • Always use unique values
  • Prefer stable IDs over array indices
  • Required when rendering lists

React State Management

Definition of State

React State Flow Diagram

State Management Overview

State refers to the data or variables that determine a component's behaviour and render information to the user.

Component Structure

In ReactJS, each component has two main aspects:

  • View: Defined using JSX or HTML components
  • Data: Stored in variables and managed through state

Data Storage Options

To hold data inside a component, you have three options:

  • Variables: Local variables within the component
  • Props: Data passed from parent components
  • State: Internal, mutable data storage

Key Point: State is the recommended way to handle dynamic data that can change over time and needs to trigger re-renders.

Key Characteristics of React State

  • Mutable through the setState() method
  • Component-specific management
  • Available in class components by default
  • Accessible in functional components via hooks (since v16.8)
  • Automatically triggers DOM re-rendering upon changes

State Management Methods

1. Local State

State stored and managed within an individual component, perfect for component-specific data.

  • Simplest form of state management
  • Ideal for component-level data
  • Uses useState or this.state

2. Context

Shares state between components without prop drilling.

  • Avoids passing props through intermediate components
  • Ideal for theme, user preferences, authentication
  • Built into React core

3. State Management Libraries

External solutions for complex state management:

  • Redux: Predictable state container
  • MobX: Simple, scalable management
  • Zustand: Lightweight and flexible
  • Recoil: Facebook's atomic state management

4. React Hooks

Built-in state management features for functional components:

  • useState: Simple state management
  • useReducer: Complex state logic
  • useContext: Context consumption
  • useEffect: Side effects handling

5. GraphQL

Complete data and state management solution:

  • Efficient data fetching with queries
  • State updates through mutations
  • Real-time updates via subscriptions
  • Client-side caching with Apollo Client

Selection Guide: Choose your state management approach based on:

  • Application size and complexity
  • Team expertise and preferences
  • Scalability requirements
  • Performance needs

React Component

const { useState } = React;

function App() {
    const [bgColor, setBgColor] = 
        useState('transparent');

    const handleClick = () => {
        setBgColor(bgColor === 'transparent' 
            ? 'red' 
            : 'transparent');
    }

    return (
        <div 
            className="container"
            style={{ backgroundColor: bgColor }}
            onClick={handleClick}
        >
            sarorAcademy (Click me!)
        </div>
    );
}

const root = ReactDOM.createRoot(
    document.getElementById('root')
);
root.render(<App />);
Key Points
  • Component-based architecture
  • State management with hooks
  • Declarative approach

Context API for State Management

Context Example:


const ThemeContext = React.createContext();

function ThemeProvider({ children }) {
    const [theme, setTheme] = useState('light');
    return (
        <ThemeContext.Provider value={{ theme, setTheme }}>
            {children}
        </ThemeContext.Provider>
    );
}

State Management Best Practices

  • Use setState() for state updates, never modify state directly
  • Keep state as minimal as possible
  • Lift state up when needed by multiple components
  • Use appropriate state management solution based on app size
  • Implement proper error boundaries for state-related errors
