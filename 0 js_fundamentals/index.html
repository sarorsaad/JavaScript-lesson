JavaScript Fundamentals Tutorial

1. Functions

What are Functions?

Functions are reusable blocks of code that perform specific tasks. They help organize code, reduce repetition, and make programs more maintainable.

Key Points:

  • Function Declaration: The basic way to define a function using the 'function' keyword
  • Function Expression: Defining a function by storing it in a variable
  • Arrow Functions: A concise way to write functions introduced in ES6
  • Parameters: Values that functions can accept and use
  • Return Value: The output that a function produces

Code:


// 1. Basic Function Declaration
function greet(name) {
    return "Hello, " + name + "!";
}

// 2. Function Expression
const add = function(a, b) {
    return a + b;
};

// 3. Arrow Function
const multiply = (x, y) => x * y;

// 4. Function with Default Parameters
function welcome(name = "Guest") {
    return `Welcome, ${name}!`;
}

2. Classes

What are Classes?

Classes are templates for creating objects that bundle data with code to work on that data. They provide a way to create reusable components and implement object-oriented programming concepts.

Key Points:

  • Constructor: Special method for initializing new objects
  • Methods: Functions that belong to the class
  • Inheritance: Creating new classes based on existing ones
  • Static Methods: Methods that belong to the class itself, not instances
  • Encapsulation: Bundling data and methods that work on that data within a single unit

Code:


// Basic Class Definition
class Car {
    constructor(brand, model) {
        this.brand = brand;
        this.model = model;
    }

    getInfo() {
        return `This is a ${this.brand} ${this.model}`;
    }

    static getCarType() {
        return "This is a vehicle";
    }
}

// Inheritance Example
class ElectricCar extends Car {
    constructor(brand, model, batteryCapacity) {
        super(brand, model);
        this.batteryCapacity = batteryCapacity;
    }

    getInfo() {
        return `${super.getInfo()} with ${this.batteryCapacity}kWh battery`;
    }
}

3. Modules

What are Modules?

Modules are a way to split JavaScript code into separate files, making it easier to maintain and organize. They allow you to export and import code between files.

Key Points:

  • Export: Making code available to other modules
  • Import: Using code from other modules
  • Default Export: The main export from a module
  • Named Exports: Exporting multiple values from a module
  • Module Scope: Variables are scoped to the module by default

Code:


// math.js - Example of a module
export const add = (a, b) => a + b;
export const subtract = (a, b) => a - b;
export const multiply = (a, b) => a * b;

// Default export
export default class Calculator {
    static square(x) {
        return x * x;
    }
}

4. Libraries

What are Libraries?

Libraries are collections of pre-written code that can be reused to perform common tasks. They save development time and provide tested, optimized solutions.

Key Points:

  • Package Management: Using npm or CDN to include libraries
  • Documentation: Understanding how to use library features
  • Version Control: Managing library dependencies
  • Common Use Cases: DOM manipulation, utilities, HTTP requests
  • Performance: Considering the impact on load times

Code:


// Examples of popular libraries

// jQuery
$("#element").hide();
$("#button").click(() => {
    alert("Button clicked!");
});

// Lodash
const numbers = [1, 2, 3, 4, 5];
_.shuffle(numbers);
_.sum(numbers);

// Moment.js
moment().format("MMMM Do YYYY");
moment().add(7, "days");

// Axios
axios.get("https://api.example.com/data")
    .then(response => console.log(response.data))
    .catch(error => console.error(error));
