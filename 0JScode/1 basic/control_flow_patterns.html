2. Parameters (optional)
3. Code block 		- Parentheses () for parameters
- Curly braces {} for code block 		function greet() { console.log("Hello!"); } Defines reusable blocks of code that can be called multiple times
Conditions if
else if
else 		1. Condition expression
2. True code block
3. False code block (optional) 		- Parentheses () for condition
- Curly braces {} for code blocks
- Comparison operators ===, >, <, etc. 		if (score >= 60) { console.log("Passed!"); } else { console.log("Failed!"); } Controls code execution based on conditions
Loops for 1. Initialization
2. Condition
3. Increment/Decrement
4. Code block 		- Parentheses () for loop definition
- Semicolons ; between parts
- Curly braces {} for code block 		for (let i = 1; i <= 5; i++) { console.log(i); } Repeats code block for a specified number of times

Key Differences from Variables:

  • Functions use () to define parameters and {} to contain code, not = for assignment
  • Conditions use () for logical expressions and {} for code blocks, evaluating true/false
  • Loops use () to define their pattern and {} for repeated code, focusing on iteration not assignment

Why No Assignment (=) Operator?

Unlike variables, control flow structures don't use the = operator because:

  • They define behavior and logic flow rather than storing values
  • They use special symbols like () and {} to define their structure
  • They execute code blocks rather than assign values
