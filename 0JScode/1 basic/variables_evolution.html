تطور المتغيرات: من JavaScript إلى React ثم Next.js

نستعرض كيفية استخدام المتغيرات في الأطر المختلفة وكيف تطورت من JavaScript الأساسية إلى React ثم Next.js

1. المتغيرات في JavaScript

في JavaScript، نستخدم var و let و const لتعريف المتغيرات. كل منها له خصائصه وحالات استخدامه.

// تعريف المتغيرات بطرق مختلفة var oldVariable = "يمكن إعادة تعريفه"; let modernVariable = "متغير حديث"; const constantValue = "لا يمكن تغييره"; // يمكن تغيير قيمة let modernVariable = "قيمة جديدة";
ملاحظة: var يمكن الوصول إليه في نطاق الوظيفة، بينما let في النطاق المحلي فقط
⚛️

2. المتغيرات في React (محاكاة)

في React، نستخدم useState لإدارة الحالة وإعادة تقديم المكونات عند تغيير القيم

import React, { useState } from 'react'; function Counter() { const [count, setCount] = useState(0); return ( <div> Count: {count} <button onClick={() => setCount(count + 1)}> زيادة العد </button> </div> ); }
العداد: 0
ملاحظة: useState يعيد مصفوفة بها القيمة ودالة لتحديثها
▲

3. المتغيرات في Next.js (محاكاة)

في Next.js، نستخدم المتغيرات مع getStaticProps و getServerSideProps لجلب البيانات

// pages/index.js export async function getStaticProps() { const res = await fetch('https://api.example.com/data'); const data = await res.json(); return { props: { data, }, }; } export default function Home({ data }) { return ( <div> {JSON.stringify(data, null, 2)} </div> ); }
انقر على الزر لمحاكاة جلب البيانات
ملاحظة: البيانات تُجلب قبل تقديم الصفحة في Next.js
