Understanding Function Syntax Differences

1. Function Declaration


// Function Declaration
function greet(name) {
    return `Hello, ${name}!`;
}

Key points:

  • Starts with function keyword
  • Has a function name (greet)
  • Can be used before it's defined (hoisted)

2. Function Expression


// Function Expression
const greet = function(name) {
    return `Hello, ${name}!`;
};

Key points:

  • Assigned to a variable
  • Function itself doesn't have a name (anonymous)
  • Must be defined before using it (not hoisted)

3. Arrow Function


// Arrow Function
const greet = (name) => {
    return `Hello, ${name}!`;
};

// Arrow Function (shorter version)
const greetShort = name => `Hello, ${name}!`;

Key points:

  • Uses arrow (=>) syntax
  • No function keyword needed
  • Shorter syntax for single parameters and single expressions
  • Different behavior with this keyword
