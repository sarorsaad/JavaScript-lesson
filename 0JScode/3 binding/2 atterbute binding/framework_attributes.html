HTML Attributes in Modern Frameworks

1. React Attributes

React uses camelCase for most attributes and has some special names.

// Basic React Component with Attributes
function Button() {
    const [isActive, setIsActive] = useState(false);

    return (
        <button
            className="my-button"           // Instead of 'class'
            onClick={() => setIsActive(!isActive)}
            disabled={!isActive}
            style={{ backgroundColor: 'blue' }}
            data-testid="test-button"      // Data attributes stay kebab-case
        >
            Click Me
        </button>
    );
}

Common React Attribute Changes:

HTML React Example
class className className="button"
for htmlFor htmlFor="input-field"
onclick onClick onClick={handleClick}
style="color: red;" style={{}} style={{ color: 'red' }}

2. Next.js Attributes

Next.js uses React's attribute system but adds some special attributes for optimization.

// Next.js Component with Special Attributes
import Image from 'next/image'
import Link from 'next/link'

function NextPage() {
    return (
        <div>
            {/* Optimized Image */}
            <Image
                src="/profile.jpg"
                alt="Profile"
                width={500}
                height={300}
                priority        // Load image with priority
                quality={75}    // Image quality
            />

            {/* Optimized Link */}
            <Link
                href="/about"
                prefetch={false}    // Control prefetching
                scroll={false}      // Disable scroll to top
            >
                About Page
            </Link>
        </div>
    );
}
Note: Next.js adds special attributes for:
  • Image optimization (width, height, priority, quality)
  • Link behavior (prefetch, scroll)
  • Script loading (strategy, onLoad)

3. Vue.js Attributes

Vue uses v-bind or : shorthand for dynamic attributes.

<template>
    <div>
        <!-- Static Attributes -->
        <button class="static-button">Static</button>

        <!-- Dynamic Attributes -->
        <button
            :class="buttonClass"
            :disabled="isDisabled"
            v-bind:id="buttonId"
            @click="handleClick"
            :style="{ backgroundColor: color }"
        >
            Dynamic
        </button>

        <!-- v-bind shorthand (:) -->
        <img :src="imagePath" :alt="imageDesc">

        <!-- v-on shorthand (@) -->
        <button @click="handleClick">Click</button>
    </div>
</template>

<script>
export default {
    data() {
        return {
            buttonClass: 'active-button',
            isDisabled: false,
            buttonId: 'main-button',
            color: 'blue',
            imagePath: '/image.jpg',
            imageDesc: 'Description'
        }
    }
}
</script>

Vue.js Special Directives:

Syntax Purpose Example
v-bind: or : Bind attributes :class="className"
v-on: or @ Event handling @click="handleClick"
v-model Two-way binding v-model="message"
v-if Conditional rendering v-if="isVisible"

Key Differences Summary

Feature React/Next.js Vue.js
Class Binding className="class-name" :class="className"
Style Binding style={{ color: 'red' }} :style="{ color: 'red' }"
Event Handling onClick={handleClick} @click="handleClick"
Conditional Attributes disabled={isDisabled} :disabled="isDisabled"
