Classes in JavaScript

1. Introduction to Classes

Classes in JavaScript are a fundamental component of Object-Oriented Programming (OOP), introduced in ECMAScript 2015. They serve as a 'blueprint' for creating objects, determining their properties and behaviors. Classes are primarily syntactic sugar over JavaScript's existing prototype-based inheritance.

When ES6 introduced Classes, it wasn't exactly a new object-oriented inheritance model. Instead, it's a syntactical sugar over the existing prototype-based inheritance. Let's compare the old and new ways:

Comparison between old prototype-based and new class-based syntax

Old Way (Pre-ES6)

var Meal = function(food) { this.food = food; }; Meal.prototype.eat = function() { return 'üòã'; };

New Way (ES6 Classes)

class Meal { constructor(food) { this.food = food; } eat() { return 'üòã'; } }

2. Class Basics

A class in JavaScript has several key components:

class Meal { constructor(food) { this.food = food; } // Prototype Method eat() { return 'üòã'; } // Static Method static utensil() { return 'ü•¢'; } }

Key Components:

  • Constructor: Special method for initializing class instances
  • Prototype Methods: Methods accessible by class instances
  • Static Methods: Methods called on the class itself

3. Class Methods

Strict Mode and Class Body

Class bodies are automatically executed in strict mode. This means:

  • Undeclared variables are not automatically global
  • Assignments that would silently fail throw errors
  • Attempts to delete undeletable properties throw errors
class Shape { constructor() { // This is automatically in strict mode this.type = "circle"; } }

Hoisting

Unlike functions, classes are not hoisted. You must declare a class before you can use it:

// ‚ùå This will throw a ReferenceError const shape = new Shape(); // Class declaration class Shape { constructor() { this.type = "circle"; } }

4. Getters and Setters

Using Getters and Setters

Getters and setters allow you to define object accessors (computed properties):

class Circle { constructor(radius) { this.radius = radius; } // Getter get area() { return Math.PI * Math.pow(this.radius, 2); } // Setter set area(area) { this.radius = Math.sqrt(area / Math.PI); } } const circle = new Circle(5); console.log(circle.area); // Returns the area circle.area = 78.54; // Sets new radius based on area

5. Static Methods

Static Methods

Static methods are called on the class itself, not on instances of the class:

class Course { constructor() { this.course = "JavaScript"; } normalMsg() { return "Learning " + this.course; } static staticMsg() { return "Welcome to the course!"; } } console.log(Course.staticMsg()); // Works const course = new Course(); console.log(course.normalMsg()); // Works // course.staticMsg(); // Would throw error

6. Inheritance

Class Inheritance

Classes can extend other classes using the extends keyword. The super() method must be called in the constructor:

class Shape { constructor(color) { this.color = color; } describe() { return `A ${this.color} shape`; } } class Circle extends Shape { constructor(color, radius) { super(color); // Call parent constructor this.radius = radius; } describe() { return `${super.describe()} with radius ${this.radius}`; } } const circle = new Circle("blue", 5); console.log(circle.describe()); // "A blue shape with radius 5"
