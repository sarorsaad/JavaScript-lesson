Next.js Basic Lessons

Step 1: Introduction to Next.js

Next.js is a React framework that enables features such as:

  • Server-side rendering (SSR)
  • Static site generation (SSG)
  • API routes
  • File-based routing
  • Built-in CSS and Sass support

Step 2: Setting Up Next.js

To create a new Next.js project, run:

npx create-next-app@latest my-next-app

This will create a new Next.js project with the following structure:

my-next-app/ ├── pages/ │ ├── _app.js │ └── index.js ├── public/ ├── styles/ ├── package.json └── next.config.js

Step 3: File-Based Routing

Next.js uses file-based routing. Files in the pages directory automatically become routes:

pages/ ├── index.js // → / ├── about.js // → /about └── posts/ ├── index.js // → /posts └── [id].js // → /posts/1, /posts/2, etc.

Step 4: Basic Next.js Page

Here's a basic Next.js page component:

// pages/index.js export default function Home() { return ( <div> <h1>Welcome to Next.js!</h1> <p>This is a basic Next.js page.</p> </div> ) }

Step 5: Data Fetching Methods

Next.js provides three main data fetching methods:

  • getStaticProps - Fetch data at build time
  • getServerSideProps - Fetch data on each request
  • getStaticPaths - Specify dynamic routes to pre-render
// Example of getStaticProps export async function getStaticProps() { const res = await fetch('https://api.example.com/data') const data = await res.json() return { props: { data, }, } }
